# 🚀 荟享生活 (hxlife) —— 高并发社交后端项目

# 项目技术文档

## 📖 项目概述

基于 Spring Cloud Alibaba 微服务架构的社交平台后端系统，实现了用户管理、社交互动、优惠券秒杀、地理位置服务等核心业务功能。

---

## 🛠️ 技术栈

| **维度**       | **技术选型**                                           |
| -------------- | ------------------------------------------------------ |
| **微服务框架** | Spring Cloud Alibaba (Nacos, Sentinel, Gateway, Feign) |
| **持久层**     | MySQL 8.0 + MyBatis-Plus                               |
| **缓存**       | Redis (ZSet, Hash, Set, String, GEO)                   |
| **消息队列**   | RabbitMQ                                               |
| **对象存储**   | 阿里云 OSS                                             |
| **工具类**     | Hutool, Lombok, JWT                                    |

---

## 🏗️ 微服务模块划分

| **模块**        | **职责**                                           |
| --------------- | -------------------------------------------------- |
| **用户模块**    | 用户登录信息管理、获取Token                        |
| **网关模块**    | 请求拦截、登录状态校验、请求路由                   |
| **API 模块**    | Feign 接口统一管理（服务间调用）                   |
| **Common 模块** | 公用工具类、实体类、通用组件                       |
| **博客模块**    | 发布博客并推送、优惠卷秒杀、点赞、店铺地理位置排序 |

### 2. 用户登录


- 登录成功后利用UUID随机数生成  Token
- 利用 **Redis** 存储 Token，并在拦截器中完成校验
- 支持 Token 续期与主动登出

### 3. 博客发布（Feed 流推模式）

- 采用 **推模式**，用户发布博客后通过 **RabbitMQ** 异步推送给所有粉丝的收件箱
- 解耦发布逻辑与推送逻辑，提升响应速度
- 粉丝拉取动态时直接从自己的信箱（Redis ZSet）中读取

### 4. 优惠券秒杀

- **Redis Lua 脚本**：扣减库存操作原子化，防止超卖
- **RabbitMQ 削峰填谷**：将秒杀请求异步化，队列慢慢消费更新数据库库存
- 提升用户体验：用户秒杀瞬间即可获得“排队中”反馈，无需等待数据库写入

### 5. 附近商铺查询

- 利用 Redis **GEO** 数据结构
- 将二维地理信息（经度、纬度）一维化存储
- 支持高效的附近商铺范围查询与距离排序

## 🚀 深度优化

### 1. 社交评论系统

| **优化点**       | **方案**                                                     |
| ---------------- | ------------------------------------------------------------ |
| **滚动分页**     | 利用 Redis ZSet 的 `score`（时间戳）实现滚动分页，解决传统 `LIMIT` 分页的数据重复与跳行问题 |
| **N+1 查询优化** | 采用“索引与数据分离”模式：ZSet 存 ID + 批量查询（Multi-Get）+ 内存 Map 聚合，将查询复杂度降至 O(1) 级别 |



### 3. 异步削峰

- **RabbitMQ** 承载非核心业务：
  - 发布博客后推送给粉丝
  - 评论发布、计数更新
  - 点赞/取消点赞
- 提升系统整体吞吐量与用户体验

### 4. 分布式异步一致性

- **消息驱动**：评论发布、点赞等操作通过 MQ 解耦
- **最终一致性保证**：
  - 重试机制 + 幂等性校验
  - 确保高并发下点赞数、评论数等统计数据的准确性

### 5. 防御性文件存储（阿里云 OSS）

| **安全措施**   | **实现方式**                            |
| -------------- | --------------------------------------- |
| **文件重命名** | UUID 唯一化命名，防止路径猜测           |
| **目录管理**   | 按日期自动分类存储（如 `2025/01/15/`）  |
| **双重校验**   | 后缀名白名单 + MIME Type 白名单双重校验 |
| **通用封装**   | 提供统一的 OSS 上传组件，降低接入成本   |



## ✅ 项目亮点总结

- **微服务架构**：基于 Spring Cloud Alibaba，服务拆分清晰，职责单一
- **高性能缓存**：Redis 多种数据结构（ZSet、GEO、Hash）深度应用
- **异步削峰**：RabbitMQ 解耦核心与非核心业务，提升吞吐量
- **最终一致性**：MQ + 重试 + 幂等，保证高并发下数据准确
- **安全防御**：OSS 双重校验 + 唯一化命名，防止恶意上传
- **分页优化**：ZSet 滚动分页，彻底解决传统分页痛点
