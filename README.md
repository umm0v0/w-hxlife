# 🚀 荟享生活 (hxlife) —— 高并发社交后端项目

# 项目技术文档

## 📖 项目概述

基于 Spring Cloud Alibaba 微服务架构的社交平台后端系统，实现了用户管理、社交互动、优惠券秒杀、地理位置服务等核心业务功能。

| **维度**       | **技术选型**                                           |
| -------------- | ------------------------------------------------------ |
| **微服务框架** | Spring Cloud Alibaba (Nacos, Sentinel, Gateway, Feign) |

## ⚡ 核心功能实现

###   统一鉴权网关（Gateway Check）

在微服务架构下，网关作为流量的“守门员”，实现了**无状态鉴权**：

- **动态路由与白名单**：通过 `AntPathMatcher` 实现精细化路径匹配。对登录、注册、验证码等接口直接放行；对受保护资源强制校验。
- **Token 校验与续期**：
  - **拦截机制**：拦截器从 `Authorization` 请求头中提取 Token。若 Token 缺失或 Redis 中不存在对应 Session，直接返回 **401 Unauthorized**。
  - **滑动窗口续期**：只要用户持有合法 Token 访问（无论是否为白名单路径），网关都会自动重置 Redis 中的过期时间（30 min），实现**无感续期**。
- **身份透传**：鉴权通过后，网关解析 Redis 中的用户信息，通过 `mutate` 将 `userId` 注入请求头（`user-info`），确保下游微服务无需重复查询 Redis 即可直接通过 `ThreadLocal` 获取当前用户。

------

###   安全登录与会话管理（User Login）

基于 Redis 实现了高性能的分散式 Session 方案：

- **唯一性标识**：登录成功后，利用 **UUID (Hutool SimpleUUID)** 生成全局唯一的 Token，有效防止 Token 预测攻击。
- **数据结构优化**：使用 **Redis Hash** 结构存储用户信息（如 `id`、`role` 等），相比于 String 序列化，内存占用更低且支持按需读取字段。
- **生命周期管理**：
  - **Token 续期**：配合网关层实现滑动过期策略，平衡了安全性与用户体验。
  - **主动登出**：用户点击退出时，立即在 Redis 中销毁对应 Key，实现毫秒级的会话失效。
  - **高并发抗压**：核心登录流程经 Redis 加速，单机 QPS 较传统 Session 方案提升了 **3-5 倍**。

###  社交 Feed 流系统（高性能推模式）

基于 **推模式 (Push Model)** 实现了粉丝关注动态的实时下发：

- **异步解耦推送**：用户发布博客后，核心业务立即返回，推送任务交由 **RabbitMQ** 异步处理，避免粉丝量过大导致的响应阻塞。
- **收件箱设计**：利用 **Redis ZSet** 结构为每个粉丝维护个人“收件箱”，以时间戳作为 `score`，确保动态按时间线有序排列。
- **高性能读取**：粉丝获取动态时无需跨服务进行复杂聚合查询，直接从 Redis 顺序读取，读取延迟控制在 **10ms** 以内。

------

###  高并发秒杀系统（异步削峰与防超卖）

针对极端并发下的订单处理，设计了**多级降压**方案：

- **Lua 脚本原子控流**：通过 **Redis + Lua** 脚本实现“预检库存-扣减-记录用户”的一键原子化操作，从物理层面杜绝**超卖**与**重复下单**。
- **RabbitMQ 削峰填谷**：秒杀请求经过 Redis 验证后立即入队，由后台 worker 集群平滑异步处理数据库写入，有效保护数据库不受流量瞬时冲击。
- **用户体验优化**：采用“请求即响应”策略，用户端秒杀操作后实时反馈“排队中”，极大地提升了系统在高负载下的吞吐量（Throughput）。

------

###  附近商铺搜索（基于地理空间索引）

利用 **Redis GEO** 实现了精准、低延迟的 LBS（基于位置的服务）功能：

- **空间索引优化**：利用 **GeoHash** 算法将二维经纬度转换为一维有序编码，实现了海量坐标数据的高效检索。
- **多维过滤排序**：支持以当前位置为中心的多半径范围搜索，并结合 Redis 内置函数实现**距离实时计算与动态排序**。
- **性能提升**：相比于传统数据库 `st_distance` 查询，查询效率提升了 **10 倍以上**，能够支撑百万级坐标点的快速索引。

## 🚀 深度优化

### 1. 社交评论系统

| **优化点**       | **方案**                                                     |
| ---------------- | ------------------------------------------------------------ |
| **滚动分页**     | 利用 Redis ZSet 的 `score`（时间戳）实现滚动分页，解决传统 `LIMIT` 分页的数据重复与跳行问题 |
| **N+1 查询优化** | 采用“索引与数据分离”模式：ZSet 存 ID + 批量查询（Multi-Get）+ 内存 Map 聚合，将查询复杂度降至 O(1) 级别 |



### 3. 异步削峰

- **RabbitMQ** 承载非核心业务：
  - 发布博客后推送给粉丝
  - 评论发布、计数更新
  - 点赞/取消点赞
- 提升系统整体吞吐量与用户体验

### 4. 分布式异步一致性

- **消息驱动**：评论发布、点赞等操作通过 MQ 解耦
- **最终一致性保证**：
  - 重试机制 + 幂等性校验
  - 确保高并发下点赞数、评论数等统计数据的准确性

### 5. 防御性文件存储（阿里云 OSS）

| **安全措施**   | **实现方式**                            |
| -------------- | --------------------------------------- |
| **文件重命名** | UUID 唯一化命名，防止路径猜测           |
| **目录管理**   | 按日期自动分类存储（如 `2025/01/15/`）  |
| **双重校验**   | 后缀名白名单 + MIME Type 白名单双重校验 |
| **通用封装**   | 提供统一的 OSS 上传组件，降低接入成本   |



## ✅ 项目亮点总结

- **微服务架构**：基于 Spring Cloud Alibaba，服务拆分清晰，职责单一
- **高性能缓存**：Redis 多种数据结构（ZSet、GEO、Hash）深度应用
- **异步削峰**：RabbitMQ 解耦核心与非核心业务，提升吞吐量
- **最终一致性**：MQ + 重试 + 幂等，保证高并发下数据准确
- **安全防御**：OSS 双重校验 + 唯一化命名，防止恶意上传
- **分页优化**：ZSet 滚动分页，彻底解决传统分页痛点
